## 0. 全局约束（摘自视觉规范 & 复盘）

| 约束项 | 规则 |
|--------|------|
| 磨砂重点卡 | 每屏最多 **2** 个 |
| 非重点卡底色 | **极轻薄**（`--subscreen-subcard-bg`） |
| 布局策略 | 指标密度动态：≤4 留白聚焦，5-10 双列网格，>10 密度优先+分区滚动 |

| 不改项 | 不改模块组件结构、不改字段绑定方式、不改设计系统变量 |

## 13. 坑点提醒（摘自协作复盘）

| 坑点 | 应对策略 |
|------|---------|
| 语义容易反向理解（"更实""更透"） | 统一使用档位名称：极轻薄/轻薄/标准 |
| 需求范围容易扩散 | 严格按本文档逐子屏执行，不跨范围 |
| 缺少可视化档位 | 使用 `--subscreen-subcard-bg` / `--subscreen-ring-bg` 变量 |
| 验收标准主观化 | 按第 12 节验收清单逐项检查 |
| Grid 子项溢出 | 所有 section-panel 设 `min-height: 0` |
| 列表/明细溢出 | 可增长区域设 `overflow: auto` |
