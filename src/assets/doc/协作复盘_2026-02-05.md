## 协作复盘（2026-02-05）

### 已确认协作规则（用户已确认）
- 重点磨砂规则：同意，`每个子屏最多 2 个磨砂重点卡`。
- 非重点卡透明度：默认使用 `极轻薄`。
- 布局优先级：按指标密度动态处理，`指标多 -> 信息密度优先`，`指标少 -> 呼吸感优先`。
- 迭代方式：`不需要先单页试改再批量`，按需求直接执行。

### 1. 背景
近期我们围绕大屏子屏做了高频视觉迭代，核心集中在：
- 卡片底色（深浅、透明度、是否磨砂）
- 子屏层次统一（行政/人资/资产 -> 业务/投资/重点项目 -> 经营/财务/资金）
- 多轮快速微调（同一页面连续 3-6 次参数迭代）

### 2. 协作中暴露的主要问题

#### 2.1 语义容易反向理解
典型例子：
- “更实一点” vs “更透一点”
- “深蓝突兀” vs “不要太浅失去层次”

结果：参数方向偶尔会反向，导致返工。

#### 2.2 需求粒度经常从“局部”扩展到“整屏”
典型路径：
- 先改 3 个子屏某类卡片
- 再扩到更多子屏统一
- 再改为只保留深色主题

结果：中途目标变化，修改范围不断放大。

#### 2.3 缺少“可视化档位”导致沟通成本高
同一诉求（透明度/磨砂强度）多轮口头微调，没有统一的档位标尺。

#### 2.4 验收标准多为感受描述，缺少硬约束
例如“空”“突兀”“太实”，但缺少：
- 重点项最多几处磨砂
- 非重点项统一 alpha 范围
- 文本/图表最小可视高度

### 3. 下一阶段建议的协作方式（可直接执行）

#### 3.1 建立视觉参数档位（强烈建议）
为以下参数固定 3 档：
- `subcard_alpha`: 极轻薄 / 轻薄 / 标准
- `ring_bg_alpha`: 极轻薄 / 轻薄 / 标准
- `frosted_count_per_screen`: 每屏最多 N 处

建议默认：
- 非重点卡：极轻薄档
- 重点卡：标准档
- 单屏磨砂重点：不超过 2 处

#### 3.2 每次改动采用“先小后大”节奏
- 你当前偏好是直接执行批量改动，不强制先单页试改。
- 如遇高不确定性，我会先用 1 条简短确认语句对齐方向再改。

#### 3.3 请求格式模板（你可直接复用）
建议你下次用这类格式发需求：

- 页面范围：`经营数据、财务管理`
- 改动类型：`仅间距+层次，不改指标`
- 重点卡规则：`每页最多2个磨砂`
- 透明度档位：`非重点=极轻薄，重点=标准`
- 验收条件：`不遮挡、不溢出、1080p 可完整展示`

#### 3.4 我这边的默认执行策略（后续按此执行）
- 先回显“我理解到的范围和不改项”
- 若存在歧义词（如“实”“透”）先做一次二选一确认
- 大改前先给“变更清单”
- 每次改完统一跑构建并回报

### 4. 已形成的有效协作经验
- 你对视觉主次判断非常明确，快速反馈很有效。
- “不要改模块和指标”这个约束非常好，显著降低风险。
- 你愿意连续微调，适合做参数化设计系统（变量驱动）。
