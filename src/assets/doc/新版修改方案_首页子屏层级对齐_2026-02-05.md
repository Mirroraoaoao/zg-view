# 新版修改方案：首页与子屏数据层级对齐（2026-02-05）

## 背景与目标
基于最新版《集团经营管理大屏数据体系规划方案 (CSV像素级复刻·完整版)》，本方案聚焦“首页 + 子屏”的**数据内容层级（L1/L2/L3）**对齐，而不仅是指标名称对齐。

本次目标：
1. 首页卡片指标与规划第2节保持一致。
2. 子屏模块的 L1/L2/L3 结构与规划第3节保持一致。
3. 消除“指标在错误模块出现”的层级漂移（尤其是资金情况、资产运营情况）。

## 差异清单（按层级）
## 一、首页层级差异
### 1) C位经营数据
- 现状：首页保留了 `经营目标达成率`（通过“目标达成进度”图）。
- 规划：第2节首页核心指标为 7 项，未包含 `经营目标达成率`（该项在第3节作为独立L1用于子屏）。
- 处理建议：首页移除该项，仅保留子屏展示。

### 2) 业务数据卡片
- 现状：首页为 `合作区重大项目招引` + 税收类 + 诉讼。
- 规划：首页首项是 `合作区/珠海市重大项目招引`（合并口径展示）。
- 处理建议：首页改为合并展示（如 `合作区 新X/累计Y；珠海 新A/累计B`）。

---

## 二、子屏层级差异
### 1) 经营数据（GeneralDetail）
- 现状：已覆盖指标，但页面以“营收与利润/贡献与产值/费用压降”组织，未显式体现 `L1: 经营目标达成率(不含重点工作，只计算经济指标)`。
- 建议：补充 L1 标识与说明文案，确保层级语义与规划一致。

### 2) 财务管理（FinanceDetail）
- 现状：`核心财务总览`、`一利五率`、`社会效益类`为并列面板，另有“核心财务总览（补充项）”。
- 规划：`L1: 核心财务总览` 下包含 `L2: 一利五率` 与 `L2: 社会效益类`。
- 建议：按“1个L1容器 + 2个L2子区块 + 核心指标列表”重组，去掉“补充项”割裂感。

### 3) 资金分析（TreasuryDetail）
- 现状：`资金情况`（资金整体情况/融资情况/有息负债余额）在资金分析页。
- 规划：这三项在CSV层级归属为 `业务数据` 模块（规划文档已注明）。
- 建议：从 `TreasuryDetail.vue` 迁出至 `ProjectDetail.vue`，Treasury页仅保留融资管理相关层级。

### 4) 投资运营（InvestmentDetail）
- 现状：有“投资运营概览”“市值趋势”等衍生展示。
- 规划：L1 仅需 `核心投资指标` 与 `投后运营与效益`，核心指标为三项。
- 建议：保留趋势图可视化，但在结构标题上显式映射 L1/L2，避免概览层级混淆。

### 5) 重点项目（KeyProjectsDetail）
- 现状：除两项核心指标外，仍有“重点项目概览”“项目进度说明”。
- 规划：仅两条主线：`对接经营调度系统（待明细）`、`重点项目完成率`。
- 建议：精简为双卡结构，描述区缩短为注释，不再形成新层级块。

### 6) 资产管理（AssetDetail）
- 现状：包含 `房产销售去化情况（按项目）`、`本年度已盘活资产规模`。
- 规划：这两项在CSV层级归属 `业务数据 -> 资产运营情况`。
- 建议：从资产页迁出这两项到业务数据页；资产页仅保留 `物业资产盘活` 与 `资产运营`。

### 7) 业务数据（ProjectDetail）
- 现状：L1 为 `产业招商/风险应对/项目建设/人力(业务口径)`。
- 规划：应为 `产业招商/风险应对/投资情况/人力`，且需吸纳：
`资金情况`、`资产运营情况`（来自资金/资产页迁移）。
- 建议：重组为 6 个 L1：
`产业招商`、`风险应对`、`投资情况`、`人力`、`资金情况`、`资产运营情况`。

### 8) 人力资源（HumanResourceDetail）
- 现状：`人员情况-薪酬效能`。
- 规划：该部分应归属 `L1: 人力成本预算执行情况`，L2 为 `各版块人均工资/各版块人效比`。
- 建议：改 L1 标题并补足层级文案；`干部管理`区域补 L2 名称 `集团中层干部基本情况`，并将 `职级分布` 调整为 `干部管理` 下的层级项。

### 9) 行政管理（AdminDetail）
- 现状：指标完整，含摘要和拆分项。
- 规划：L1 为 `领导行程`、`督办事项完成率`、`三会议召开次数`。
- 建议：维持现有内容，补充 L1 标记一致性即可。

## 验收标准（层级对齐）
1. 首页 9 张卡指标口径与规划第2节一致，且无额外指标。
2. 每个子屏能清晰映射到规划第3节 L1/L2/L3。
3. 资金情况与资产运营情况出现在业务数据页，不再跨页漂移。
4. 指标名称与规划“★对应指标项”逐字一致。
5. `npm run build` 通过，且首页与9个子屏手工核验通过。
